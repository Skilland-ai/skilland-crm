# How To Use The Twenty Workflows Capability

## For designing a workflow

1. Use the `twenty-workflows` capability team.
2. Read first:
   - `shared/knowledge/twenty-workflows/2026-06-07_workflow_domain_model.md`
   - `shared/knowledge/twenty-workflows/2026-06-07_api_mcp_capabilities.md`
   - `shared/knowledge/twenty-workflows/examples/ia_mujeres_workflow_patterns.md`
3. Apply skill:
   - `shared/skills/twenty-workflows/twenty-workflow-design/SKILL.md`
4. Pass safety review before implementation.

## For implementing a workflow

Default path:

1. create workflow shell or draft copy
2. update trigger
3. create steps
4. update steps
5. create edges and positions
6. read back the version
7. only then consider activation

Rules:

- use API/MCP-first
- do not use UI unless `2026-06-07_ui_last_resort.md` says it is truly required
- default to draft mode or isolated `TEST -` workflow copies
- do not activate production workflows without explicit user confirmation

## For smoke testing

1. Use:
   - `shared/skills/twenty-workflows/twenty-workflow-smoke-test/SKILL.md`
2. Reuse the test-isolation pattern from:
   - `shared/knowledge/twenty-workflows/2026-06-07_workflow_testing_and_debugging.md`
3. Query `workflowRuns` and affected records directly.
4. Cleanup is mandatory.

## For retaking Phase 4.1

Read these first:

1. `03_specs/now/004_ia_mujeres_crm_smoke_test.md`
2. `04_outputs/ia_mujeres_smoke_test/2026-06-07_workflow_test_report.md`
3. `shared/knowledge/twenty-workflows/examples/ia_mujeres_workflow_patterns.md`
4. `shared/knowledge/twenty-workflows/2026-06-07_api_mcp_capabilities.md`

Recommended sequence:

1. Re-validate current workflow IDs and statuses for the IA Mujeres test workflows.
2. Confirm whether WF-2 and WF-3 should be updated in place or recreated as new `TEST -` copies.
3. Re-run API research if anyone claims a missing step is UI-only.
4. Execute only against the isolated test deal:
   - `campaignName = TEST - IA Mujeres 2026`
   - `testMode = true`
5. Prefer no-email smoke tests.
6. Query latest `workflowRuns`, created tasks, and opportunity fields.
7. Reset the deal and remove test tasks.
8. Write a fresh report before proposing any production move.

## What not to do

- do not start in the UI by default
- do not activate broad `DATABASE_EVENT` workflows without a first-step guard
- do not rely on memory of old workflow limitations
- do not send email as part of workflow validation unless explicitly requested
