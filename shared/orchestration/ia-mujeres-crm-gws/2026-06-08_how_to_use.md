- vista `IA Mujeres — Funnel`;

## Bootstrap portable en VPS

Los scripts IA Mujeres que hablan con Twenty ya no deben depender de `/home/reboot/.claude.json`.

Orden portable de credenciales de servicio:

1. `TWENTY_API_KEY` y opcional `TWENTY_BASE_URL` por entorno
2. `TWENTY_CREDENTIALS_FILE=/absolute/path/to/twenty.json`
3. `~/.config/skilland/twenty.json`
4. `~/.claude.json`
5. `/home/reboot/.claude.json` solo como fallback legacy de compatibilidad

Shape esperada de `twenty.json`:

```json
{
  "TWENTY_API_KEY": "<secret>",
  "TWENTY_BASE_URL": "https://crm.skilland.ai"
}
```

Si hace falta auth de usuario para workflows TEST, el password no va en `scripts/.env`.

Fuentes soportadas para `TWENTY_CRM_PASSWORD`:

1. `--password <secret>` como override puntual
2. `TWENTY_CRM_PASSWORD` por entorno
3. `TWENTY_USER_CREDENTIALS_FILE=/absolute/path/to/twenty-user.json`
4. `~/.config/skilland/twenty-user.json`

Shape esperada de `twenty-user.json`:

```json
{
  "TWENTY_CRM_PASSWORD": "<secret>"
}
```

## Operación diaria segura

```bash
node scripts/ia_mujeres_operator_harness.mjs --action=status
node scripts/ia_mujeres_operator_harness.mjs --action=prepare-next-batch --limit=5
```

Estos comandos:

- audita CRM;
- selecciona siguiente tanda;
- genera payloads locales;
- no muta CRM;
- no crea Gmail drafts;
- no envía emails.

El harness es ahora la puerta de entrada recomendada. El runner `ia_mujeres_daily_operator.mjs` queda como compatibilidad/dry-run histórico.

## Harness operativo

```bash
node scripts/ia_mujeres_operator_harness.mjs --action=status
node scripts/ia_mujeres_operator_harness.mjs --action=prepare-next-batch --limit=5
node scripts/ia_mujeres_operator_harness.mjs --action=create-drafts --batch-id=<id> --apply --confirm-create-external-drafts
node scripts/ia_mujeres_operator_harness.mjs --action=send-batch --batch-id=<id> --apply --confirm-send-approved-drafts
node scripts/ia_mujeres_operator_harness.mjs --action=launch-approved-batch --batch-id=<id> --apply --confirm-create-external-drafts --confirm-send-approved-drafts
node scripts/ia_mujeres_operator_harness.mjs --action=sync-signals --apply
node scripts/ia_mujeres_operator_harness.mjs --action=reconcile-tasks --apply
node scripts/ia_mujeres_operator_harness.mjs --action=weekly-report
node scripts/ia_mujeres_operator_harness.mjs --action=email-weekly-report --apply --confirm-send-weekly-report
```

## Reglas de autorización

- `send-approved` está bloqueado.
- Todo cambio CRM requiere `--apply`.
- Todo envío externo requiere aprobación humana explícita.
- No usar outputs antiguos como revisión vigente; usar batch `2026-06-08T00-34-36-009Z` o generar uno nuevo.

