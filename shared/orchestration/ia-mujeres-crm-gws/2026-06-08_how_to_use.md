# IA Mujeres CRM/GWS — How To Use

## Experimento 0

Crear y verificar draft interno:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs --create-draft
node scripts/ia_mujeres_experiment_00_gws_lab.mjs --verify-draft --draft-id=<draft_id>
```

Enviar solo tras revisar el draft en Gmail:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs \
  --send \
  --draft-id=<draft_id> \
  --confirm-internal-send
```

Comprobar recepcion, respuesta y bounce:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs \
  --check-reception \
  --check-replies \
  --check-bounce \
  --thread-id=<thread_id>
```

Enviar una respuesta interna controlada desde `sales@reboot.academy`, solo para laboratorio:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs \
  --send-internal-reply \
  --thread-id=<thread_id> \
  --confirm-internal-reply
```

Limpiar un draft no enviado:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs \
  --delete-draft \
  --draft-id=<draft_id>
```

## Tanda real futura

Antes de una tanda externa, el operador debe exigir:

- Experimento 0 enviado y recibido. Estado actual: OK.
- Respuesta interna detectada por `thread_id`.
- Adjunto corto validado visualmente.
- Firma validada.
- Eventos locales en `events.ndjson`.
- Mapeo CRM definido para `thread_id -> deal`.
- Autorizacion humana explicita para cada tanda.

Preparar seleccion en dry-run:

```bash
node scripts/ia_mujeres_batch_runner.mjs --prepare-next-batch --limit=5
```

Este runner solo genera plan/revision. No crea drafts y no envia emails.

## Reporte semanal

Usar como fuentes:

- Twenty CRM: opportunities de `campaignName = IA Mujeres 2026`.
- GWS events: `events.ndjson`.
- Gmail polling: replies y bounces por hilos conocidos.
- Tareas/reuniones CRM: estado humano de conversaciones.

Generar reporte local:

```bash
node scripts/ia_mujeres_weekly_report.mjs --week=2026-06-08
```

El script genera Markdown y HTML; no envia email ni muta CRM.
