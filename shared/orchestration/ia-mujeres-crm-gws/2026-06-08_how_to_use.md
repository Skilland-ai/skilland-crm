# IA Mujeres CRM/GWS — How To Use

## Experimento 0

Crear y verificar draft interno:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs --create-draft
node scripts/ia_mujeres_experiment_00_gws_lab.mjs --verify-draft --draft-id=<draft_id>
```

Enviar solo tras revisar el draft en Gmail:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs \
  --send \
  --draft-id=<draft_id> \
  --confirm-internal-send
```

Comprobar recepcion, respuesta y bounce:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs \
  --check-reception \
  --check-replies \
  --check-bounce \
  --thread-id=<thread_id>
```

Limpiar un draft no enviado:

```bash
node scripts/ia_mujeres_experiment_00_gws_lab.mjs \
  --delete-draft \
  --draft-id=<draft_id>
```

## Tanda real futura

Antes de una tanda externa, el operador debe exigir:

- Experimento 0 enviado y recibido.
- Respuesta interna detectada por `thread_id`.
- Adjunto corto validado visualmente.
- Firma validada.
- Eventos locales en `events.ndjson`.
- Mapeo CRM definido para `thread_id -> deal`.
- Autorizacion humana explicita para cada tanda.
